## Git

- Инструмент контроля версий, работает в основном через командную строку (терминал).
    
- Нет отдельного ярлыка с графическим интерфейсом при установке.
    
- Можно запускать через терминал или Git Bash.
    
- Используется для управления кодом, версиями, совместной работы.
    

## Git Bash

- Это эмулятор терминала с поддержкой Git-команд в Windows.
    
- Ярлык появляется в меню Пуск.
    
- Удобно запускается и используется для ввода команд Git в среде, похожей на Linux shell.
    
- Не имеет отдельного графического интерфейса, только консоль.
    

## GitHub Desktop

- Это самостоятельное приложение с полным графическим интерфейсом (GUI).
    
- В меню Пуск появляется ярлык, запускается как обычная Windows-программа.
    
- Позволяет работать с Git и GitHub через удобные окна и кнопки, не используя командную строку.
    
- Упрощает базовые операции с репозиториями, коммитами, ветками, а также синхронизацию с GitHub.com.
    

## Conda

- Менеджер пакетов и сред, работает в командной строке.
    
- При установке появляется возможность использовать Anaconda Navigator — приложение с GUI для управления средами (если установлен пакет Anaconda, а не только Miniconda).
    
- В противном случае активировать и управлять средами можно через терминалы (Git Bash, PowerShell).
    

## Node.js

- Среда выполнения JavaScript.
    
- После установки появляется командная строка `node` и пакетный менеджер `npm`.
    
- Нет отдельного графического интерфейса, управляется через терминал.
    
- Можно запускать JavaScript-файлы и устанавливать пакеты в терминале.
    

## FFmpeg

- Командный пакет для обработки видео и аудио.
    
- Не имеет графического интерфейса (без установки сторонних GUI).
    
- Управляется через терминал с помощью команд.

## Краткий обзор, куда обычно устанавливаются перечисленные инструменты и почему:

| Инструмент         | Тип установки           | Где устанавливается по умолчанию                                                   | Почему именно туда устанавливается                                                 |
| ------------------ | ----------------------- | ---------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| **Git**            | Системный пакет         | Обычно в папку Program Files\Git или Program Files (x86)\Git на системном диске C: | Стандартное место для системных утилит, чтобы git был доступен из любого терминала |
| **Git Bash**       | Консольная оболочка Git | В том же пакете Git, в папке Program Files                                         | Оболочка вместе с Git для удобства работы с командами                              |
| **GitHub Desktop** | Графический клиент      | Program Files\GitHub Desktop (C:\Program Files)                                    | Как обычное приложение для Windows, стандартный путь для ПО                        |
| **Conda/Anaconda** | Менеджер пакетов и сред | По умолчанию в C:\Users<Имя_пользователя>\Anaconda3                                | В пользовательской директории для изоляции от системных прав                       |
| **Node.js**        | Среда выполнения JS     | Program Files\nodejs (на C:)                                                       | Стандартное место для программ, чтобы система могла найти node и npm               |
| **FFmpeg**         | Консольные утилиты      | Обычно в Program Files\ffmpeg или в пользовательской папке                         | Для удобства доступа из командной строки и системных путей                         |
| **Visual Studio**  | Полноценная IDE         | Program Files\Microsoft Visual Studio                                              | Крупное специализированное ПО ставится в стандартные системные папки               |
| **CUDA Toolkit**   | Разработка на GPU       | Program Files\NVIDIA GPU Computing Toolkit\CUDA\vX.X                               | NVIDIA стандартизировала этот путь для совместимости с ПО и переменными окружения  |

## Почему не предлагают выбирать папку?

- Указанные пути — стандарты, обусловленные требованиями разработчиков для корректной работы, поиска библиотек, DLL и инструментов.
    
- Менять путь можно, но это требует дополнительной настройки переменных окружения и путей в IDE.
    
- Для большинства пользователей рекомендуется использовать пути по умолчанию, чтобы избежать ошибок и сложностей.
    

## Кратко для чего нужны эти установки отдельно от Pinokio

- Инструменты разработки (Git, Conda, Visual Studio, CUDA) нужны для программирования, поддержки проектов и запуска кода.
    
- Node.js — для запуска JavaScript приложений и серверных процессов.
    
- FFmpeg — отдельный аудио-видео движок для конвертации и обработки.
    
- GitHub Desktop — удобное графическое управление репозиториями.
    
- Все они стандартно используемые инструменты вне зависимости от Pinokio, полезны для разработки, научных расчетов, мультимедиа.
    

Если нужно, могу помочь с проверкой путей и переменных окружения или объяснить, как управлять этими инструментами вручную.Все перечисленные инструменты устанавливаются в стандартные системные или пользовательские папки по умолчанию, чтобы обеспечить совместимость и удобство:

- Git и Git Bash обычно в Program Files\Git на диске C, чтобы быть доступными из командной строки любой части системы.
    
- GitHub Desktop как обычное Windows-приложение в Program Files\GitHub Desktop.
    
- Conda/Anaconda ставится в C:\Users<имя пользователя>\Anaconda3 для изоляции среды без системных прав.
    
- Node.js в Program Files\nodejs, чтобы терминал находил его команды.
    
- FFmpeg обычно в Program Files\ffmpeg или в пользовательской папке, чтобы его можно было запускать из командной строки.
    
- Visual Studio, мощная IDE, инсталлируется в Program Files\Microsoft Visual Studio.
    
- CUDA Toolkit лежит в Program Files\NVIDIA GPU Computing Toolkit\CUDA\vX.X — такой путь стандартизирован NVIDIA для совместимости.
    

Не предлагают менять папку, чтобы избежать путаницы с настройками путей и переменных окружения. Это упрощает корректную работу и установку.

Эти инструменты служат для разработки, управления версиями, запуска кода, обработки видео/аудио, работы с GPU и нужны не только для Pinokio, а для любых проектов и задач.
