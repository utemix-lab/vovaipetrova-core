
Git — это система контроля версий (Version Control System, VCS), которая позволяет отслеживать все изменения в файлах проекта, управлять разными версиями, а также эффективно работать над проектом нескольким разработчикам одновременно. Git хранит снимки файлов (а не просто изменения), что позволяет быстро восстанавливать любую предыдущую версию и избегать конфликтов при совместной работе.

---

## Чем отличаются Git, Git Bash и GitHub Desktop?

|Инструмент|Описание|Назначение|
|---|---|---|
|**Git**|Сам инструмент — система контроля версий с командной строкой и библиотеками.|Установка Git даёт команды для управления репозиториями, ветками и историями изменений.|
|**Git Bash**|Эмулятор командной строки для Windows, который предоставляет Unix-подобное окружение.|Позволяет вводить Git-команды в удобном терминале с поддержкой bash-скриптов.|
|**GitHub Desktop**|Графический клиент для Git с визуальным интерфейсом, издаваемый GitHub.|Упрощает работу с Git через GUI, позволяет работать с репозиториями GitHub без командной строки.|

---

## Итого

- Git — это фундаментальный инструмент, который вы запускаете через командную строку (например, в Git Bash).
    
- Git Bash облегчает работу с Git на Windows через доступный терминал.
    
- GitHub Desktop — пользовательский интерфейс, который скрывает команды Git под кнопками и окнами.
    

Для работы с Git и проектами (включая установку Pinokio и Open Audio) Git как система обязательна, Git Bash — удобный командный интерфейс, а GitHub Desktop — опциональный графический клиент.
