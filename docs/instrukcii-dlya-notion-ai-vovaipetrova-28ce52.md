# Инструкции для Notion AI — vova_i_petrova

### Цель

Единая памятка для Notion AI: где «истина», как хранить и обновлять знания, что делать при сбросе сессии.

### Где хранится истина

- Основной источник: GitHub репозиторий vovaipetrova-core (папка docs/)
- Формат: Markdown и CSV «как есть» из экспорта Notion

### Мини‑конвенции контента

- Имена файлов: kebab-case
- Первый заголовок в файле: # Заголовок
- В конце блока — раздел "Связано с…" с относительными ссылками на соседние материалы
- Изображения/таблицы — рядом с соответствующим .md

### Рабочий поток (минимум)

1) В Notion: Export → Format: Markdown & CSV → Include subpages

2) Распаковать ZIP локально

3) Скопировать содержимое экспорта в папку docs/ репозитория

4) Commit → Push

### План по шагам

- Шаг 1: Держать экспорт Notion «как есть» в docs/
- Шаг 2: Навести порядок в именах и заголовках ключевых файлов
- Шаг 3: Добавить оглавления и связи (README в docs/ и по разделам)
- Шаг 4: Подключить витрину (Hugo/WordPress) и/или индексацию для LLM

### Что делать при сбросе сессии

- Опираться на эту страницу и на GitHub (docs/)
- При необходимости повторить: экспорт из Notion → копирование в docs/

### Дополнительно (позже)

- GitHub Actions для автоматической проверки ссылок и лёгких преобразований
- Возможная индексация для LLM (векторное хранилище) — после стабилизации структуры

### Идентификатор проекта

- Каноническое имя (slug): `vova_i_petrova`
- Правила написания: нижний регистр, символы [a–z], цифры, `_` как разделитель
- Применение: имена веток/папок/файлов, теги в YAML/JSON, переменные окружения

### Конвенции экспорта (готовность в любой момент)

- Корневая страница для экспорта в Notion: «Вова и Петрова»
- Всегда включаем Include subpages при экспорте
- Структура в Notion: все ключевые разделы — подстраницы «Вова и Петрова»
- В репозитории: весь экспорт складываем «как есть» в папку `docs/`
- Имена файлов: kebab-case (`primer-fayla.md`). Если нужен стабильный slug — используем `vova_i_petrova` как префикс, либо поле в фронт-маттере позже
- В каждом .md — H1 вверху; внизу, по возможности, блок «Связано с…»

### Чек‑лист перед экспортом

- [ ]  Материалы, которые должны попасть в ZIP, находятся внутри «Вова и Петрова»
- [ ]  Нет приватных токенов/секретов в тексте
- [ ]  Экспорт: Markdown & CSV + Include subpages
- [ ]  Загрузка ZIP в `uploads/` и запуск GitHub Actions «Notion import to docs»
