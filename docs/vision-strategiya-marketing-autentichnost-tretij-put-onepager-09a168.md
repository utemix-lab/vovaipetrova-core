# Vision — Стратегия: Маркетинг × Аутентичность × Третий путь (one‑pager)

### Резюме

Мы строим продукт и коммуникацию на управляемом напряжении между «Маркетингом» (институциональное доверие) и «Аутентичностью» (личное доверие), добавляя третий путь: Тихая комната, Радикальная прозрачность, Соучастие. Источник истины — Notion; GitHub — зеркало и автопилот.

### Три опоры ценности

- Маркетинг как институция: стабильность, качество, гарантия результата, единый язык.
- Аутентичность как человек: голос, история, дневник решений (ADR) и хода работ.
- Третий путь: Тихая комната (минимализм, польза), Радикальная прозрачность (открытые процессы), Соучастие (пользователь‑соавтор).

### Операционная модель

- Источник истины: Notion (vision, ADR, specs, playbook).
- Зеркало/исполнение: GitHub (docs, prototype, workflows), Actions как автопилот.
- Контекстный пакет: notion-brain для любых исполнителей (LLM/людей).
- Конвейер: Vision → ADR → Spec → Scaffold → PR → Normalize/Lint → Preview/Deploy.

### Продукт V1 (каркас)

- Шапка с ролями и корзина артефактов (без сервера, Pages + LocalStorage).
- Таксономия: theme, action, product, tool, role, country/producer.
- Контент: Markdown + front matter, видимые #Дизайн и #Проектирование, скрытые machine_tags.

### Позиционирование и тон

- Тихая комната: спокойный UI, польза > шум.
- Прозрачная мастерская: объясняем решения и ограничения.
- Соучастие: простые действия → от зрителя к соавтору.

### Метрики

- Новичок: время до целевого клика < 10 сек; ≥ 2 CTA клика за сессию.
- Клиент: ≥ 10% сессий «корзина → черновик заявки».
- Dev: время до первого PR < 1 час после чтения notion‑brain; ≥ 80% PR без замечаний линтера.
- Системные: 100% валидных front matter; деплой прототипа < 5 мин.

### Риски и меры

- Инфляция доверия → публичные ADR и проверяемые метрики.
- Переусложнение пайплайна → минимальный notion‑brain, автоматизацию включаем по мере стабилизации.
- Раздвоение «места правды» → правим в Notion, GitHub — зеркало.

### Дорожная карта (кратко)

- Спринт 1: V1 шапка + роли + корзина; нормализация импорта; ручной Pages‑деплой.
- Спринт 2: Поиск с фасет‑чипами, /request, первые кейсы.
- Спринт 3: content‑lint, scaffold‑CLI, страны/производители.
- Спринт 4: Публичные «эпизоды» (радикальная прозрачность).

### Роль ИИ

- Архитектор (в Notion): смыслы, рамки, критерии.
- Исполнители (Cursor/Aider/Windsurf): масштабируемые правки и сборка.
- Принцип этики: никакой «поддельной» искренности; аутентичность = объяснение решений.
