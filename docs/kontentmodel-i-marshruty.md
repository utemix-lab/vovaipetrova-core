# Контент‑модель и маршруты

## TL;DR
- Определяем разделы, корневые URL и их аудиторию.  
- Карта маршрутов учитывает корзину артефактов и заявки.  
- Роль пользователя влияет на подсказки и приоритеты навигации.  
- Документ является основой для генерации sitemap и линтера.

## Основные разделы
- `/` — главная, тизер портфолио, CTA «Сгенерировать заявку».  
- `/services` — каталог услуг с кнопкой «+ в артефакты».  
- `/portfolio` и `/portfolio/[slug]` — кейсы и детальные страницы.  
- `/kb` — база знаний с поиском и фасетами.  
- `/think-tank` — навигация для роли Dev.  
- `/studio` — рабочая зона генерации, доступна по роли.  
- `/auth` — вход/регистрация.

## Артефакты и заявка
- `/artifacts` — список артефактов (гость/аккаунт), фильтры по типу и фасетам.  
- `/artifacts/picker` — шторка выбора прямо из контента.  
- `/request` — форма заявки: текст, контакт, автоматическая подстановка артефактов.  
- `/services` и `/cases` добавляют элементы в корзину через CTA.

## Роли и режимы
- Параметр `role` в cookie/сессии настраивает подсказки и быстрые кнопки.  
- Роль `novice` видит дорожную карту, `client` — услуги и заявку, `dev` — Think Tank.  
- Навигационные блоки синхронизируются с `navigaciya-*.md`.

## Компоненты UI
- Header, RoleSwitch, NavBar, Breadcrumbs, FacetChips, ArtifactButton.  
- Каждый компонент документируется в `prototype/components/*`.  
- Метаданные (title, description, og) хранятся рядом с маршрутом.
