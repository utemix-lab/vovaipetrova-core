# ADR — Source of truth и зеркалирование

### Заголовок

Источник истины — Notion. Репозиторий — зеркало для исполнения и публикации.

### Контекст

- Ранний этап требует скорости вносить массовые правки и фиксировать архитектуру
- Нужны предпросмотры (Pages) и истории изменений (Git)

### Решение

- Все архитектурные решения и правила ведём в Notion
- Экспортируем компактный пакет notion-brain/ в репозиторий (ADR/Specs/Instructions/context-map)
- Контент и прототип живут в репо (docs/, prototype/). Конвейер импорта Notion → docs включается по мере надобности
- Actions: раздельные workflow для импортов и деплоя

### Последствия

- Плюсы: одна точка мышления (Notion), прозрачная история и предпросмотр (GitHub)
- Риски: рассинхронизация. Митигируем чек‑листами и явными экспортами только notion-brain
- Масштабирование: любые LLM‑исполнители читают notion-brain как контекст

### Статус

Accepted (V1)
