# Tags

## TL;DR

Tags (теги) — это система категоризации контента через видимые теги и машинотеги. Используется для навигации, фильтрации и поиска материалов в базе знаний.

## Что это

Система тегов состоит из двух уровней:

### Видимые теги (`tags`)
- Формат: TitleCase с подчёркиваниями (`#Проектирование`, `#Видео`)
- Отображаются пользователю в карточках и на страницах
- Используются для визуальной категоризации

### Machine tags (`machine_tags`)
- Формат: `category/value` (`theme/automation`, `action/build`)
- Не отображаются напрямую
- Используются для автоматической фильтрации и поиска

## Правила назначения

1. **Минимум 3 тега**: Каждая страница должна иметь минимум 3 тега
2. **Группы**: По одному тегу из трёх групп (тема, действие, технология)
3. **Нормализация**: Теги нормализуются через `tags.yaml` и aliases

## Хранение

### В документах

```yaml
tags:
  - Проектирование
  - Видео
machine_tags:
  - theme/automation
  - action/build
```

### В словаре

`docs/nav/tags.yaml` содержит:
- Канонические значения тегов
- Aliases (синонимы)
- Маппинг видимых тегов на machine tags

## Использование

### В Explorer

Теги используются для:
- Фильтрации по категориям
- Поиска связанных материалов
- Группировки контента

### В автолинкинге

Aliases из `tags.yaml` используются для автолинкинга упоминаний терминов.

## Команды

```bash
# Проверка тегов
npm run normalize --tags-only

# Линтинг тегов
npm run lint:docs
```
