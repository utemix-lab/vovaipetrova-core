# Autolink

## TL;DR

Autolink (автолинкинг) — это автоматическое преобразование упоминаний терминов в ссылки на соответствующие страницы. Использует словарь slug/aliases и безопасные правила для предотвращения ложных совпадений.

## Что это

Autolink:
- Находит упоминания терминов в тексте
- Преобразует их в Markdown ссылки на соответствующие страницы
- Игнорирует упоминания в code blocks и существующих ссылках

## Правила безопасного линкинга

1. **Точные совпадения**: Работает только для точных совпадений по `canonical_slug` и `title`
2. **Границы слова**: Срабатывает только на целых словах (word boundaries)
3. **Игнорирование контекста**: Пропускает code blocks и существующие ссылки
4. **Приоритет**: Использует страницу с `canonical_slug` при конфликтах

## Использование

### Dry run (проверка)

```bash
npm run autolink:dry
```

### Автолинкинг всех файлов

```bash
npm run autolink
```

### Конкретный файл

```bash
node scripts/autolink.mjs --file docs/some-file.md
```

## Примеры

**Before:**
```markdown
Мы используем Flux для генерации видео.
```

**After:**
```markdown
Мы используем [Flux](flux.md) для генерации видео.
```

## Технические детали

- **Словарь**: Строится из `prototype/data/pages.json` и `docs/nav/tags.yaml`
- **Обработка**: Markdown файлы обрабатываются с сохранением front matter
- **Производительность**: Обработка всех файлов занимает < 5 секунд
