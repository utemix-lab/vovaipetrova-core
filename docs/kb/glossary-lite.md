## A

### Aliases

Синонимы и альтернативные названия для терминов, тегов и страниц. Используются для нормализации и автолинкинга.

→ [Читать карточку](kb/aliases.md)

### Autolink

Автоматическое преобразование упоминаний терминов в ссылки на соответствующие страницы. Использует slug/aliases map и безопасные правила линкинга.

→ [Читать карточку](kb/autolink.md)

<a id="b"></a>

## B

### Backlinks

Входящие ссылки на страницу — список всех страниц, которые ссылаются на текущую. Используется для навигации и понимания связей между документами.

→ [Читать карточку](kb/backlinks.md)

<a id="c"></a>

## C

### Canonical Slug

Канонический идентификатор страницы для ссылки на документ. Обеспечивает стабильность ссылок при переименованиях.

→ [Читать карточку](kb/canonical-slug.md)

<a id="f"></a>

## F

### Facets

Фасетная модель навигации: темы, действия, продукты, технологии и роли. Используется для фильтрации и категоризации контента.

→ [Читать карточку](kb/facets.md)

## L

### Link-map

Карта маппинга проблемных ссылок на канонические страницы. Используется для разрешения broken links и нормализации ссылок.

→ [Читать карточку](kb/link-map.md)

<a id="m"></a>

## M

### Machine Tags

Машиночитаемые теги для автоматической категоризации и фильтрации контента. Используются в фасетной модели навигации.

→ [Читать карточку](kb/machine-tags.md)

<a id="r"></a>

## R

### Routes

Маршруты навигации сайта, определённые в routes.yml. Связывают URL-пути с документами и определяют структуру сайта.

→ [Читать карточку](kb/routes.md)

## S

### Snapshot

→ [Читать карточку](kb/snapshot.md)
