# Canonical Slug

## TL;DR

Canonical slug — это канонический идентификатор страницы, который используется как основной способ ссылки на документ. Обеспечивает стабильность ссылок при переименованиях и изменениях структуры.

## Что это

Canonical slug:
- Определяется в front matter документа как `slug:`
- Используется для генерации URL страницы
- Является приоритетным при автолинкинге и разрешении ссылок

## Правила

1. **Формат**: kebab-case (строчные буквы, дефисы вместо пробелов)
2. **Уникальность**: Каждый slug должен быть уникальным в рамках проекта
3. **Стабильность**: Не должен изменяться без необходимости

## Использование

### В front matter

```yaml
---
title: Название страницы
slug: nazvanie-stranicy
---
```

### В автолинкинге

При автолинкинге терминов используется `canonical_slug` как приоритетный идентификатор для разрешения конфликтов.

### В link-map

Link-map использует canonical slug как целевое значение для маппинга проблемных ссылок.

## Разрешение конфликтов

При дублировании slug:
- Добавляется суффикс с хешем: `slug-hash123`
- Или используется полный путь: `docs/section/slug.md`
