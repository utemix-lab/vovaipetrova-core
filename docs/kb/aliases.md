# Aliases

## TL;DR

Aliases (синонимы) — это альтернативные названия для терминов, тегов и страниц, которые используются для нормализации контента и автоматического линкинга.

## Что это

Aliases позволяют:
- Нормализовать различные варианты написания одного термина
- Автоматически преобразовывать упоминания в ссылки на канонические страницы
- Поддерживать поиск по синонимам

## Где используются

### В `tags.yaml`

```yaml
aliases:
  "UX": ["theme/ux"]
  "Генерация_Видео": ["theme/video"]
```

### В автолинкинге

Скрипт `scripts/autolink.mjs` использует aliases из `tags.yaml` и `pages.json` для поиска соответствий при автолинкинге терминов.

### В нормализации

Скрипт `scripts/normalize.mjs` использует aliases для приведения тегов к каноническим значениям.

## Правила

1. **Каноническое значение**: Один термин имеет одно каноническое значение (canonical)
2. **Синонимы**: Все варианты написания указываются как aliases
3. **Регистр**: Aliases могут быть в любом регистре, но canonical всегда в стандартном формате

## Примеры

- `UX` → `theme/ux`
- `Генерация_Видео` → `theme/video`
- `Flux` → страница `flux.md`
