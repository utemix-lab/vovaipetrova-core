# Briefs UX — доводки представлений By Lane / Pending Review / Ready for Copilot

## Цель

Улучшить UX представлений Briefs в Notion для более эффективной работы с задачами: упростить навигацию, сделать информацию более доступной и улучшить приоритизацию.

## Область изменений

Только представления (Views) в базе данных Briefs. Изменения выполняются вручную в Notion UI.

## Представление 1: By Lane (Board)

### Текущее состояние

- **Тип**: Board
- **Группировка**: по `Lane`
- **Сортировка**: по `SLA` (ascending)

### Рекомендуемые улучшения

#### 1. Видимые поля (Properties)

**Минимальный набор:**
- `Title` (всегда видим)
- `Status` (для быстрого понимания состояния)
- `SLA` (для контроля сроков)
- `Executor` (для понимания, кто выполняет)
- `Overdue` (для выделения просроченных задач)

**Расширенный набор (опционально):**
- `Lane` (для подтверждения группировки)
- `Requires Review` (для понимания необходимости ревью)
- `RFC Link` (для быстрого доступа к связанным RFC)

#### 2. Сортировка

**Основная сортировка:**
- По `SLA` (ascending) — сначала задачи с ближайшим сроком

**Вторичная сортировка (если поддерживается):**
- По `Status` (custom order: Ready → In progress → Review → Done)
- По `Created time` (descending) — новые задачи внизу

#### 3. Фильтры

**Базовые фильтры:**
- `Status` is not `Done` (скрыть завершенные задачи)
- `Status` is not `Cancelled` (скрыть отмененные задачи)

**Опциональные фильтры:**
- `Overdue` is `Yes` (для выделения просроченных задач)
- `Lane` is one of {Infra, Docs, IA} (для фокуса на конкретных дорожках)

#### 4. Группировка

**Основная группировка:**
- Group by: `Lane`

**Опциональная вторичная группировка (если поддерживается):**
- Group by: `Status` (внутри каждой Lane)

#### 5. Визуальные улучшения

- **Цветовое кодирование карточек:**
  - Красный/оранжевый — просроченные задачи (`Overdue = Yes`)
  - Желтый — задачи с приближающимся SLA (менее 24 часов)
  - Зеленый — задачи с достаточным запасом времени

- **Иконки статусов:**
  - Ready — зеленая галочка
  - In progress — синий индикатор
  - Review — желтый предупреждающий знак
  - Overdue — красный восклицательный знак

## Представление 2: Pending Review (Table)

### Текущее состояние

- **Тип**: Table
- **Фильтры**: `Status = Review` AND `Requires Review = Checked`

### Рекомендуемые улучшения

#### 1. Видимые поля (Properties)

**Обязательные поля:**
- `Title` (всегда видим)
- `Status` (для подтверждения статуса Review)
- `SLA` (для контроля сроков ревью)
- `Overdue` (для выделения просроченных ревью)
- `Lane` (для понимания контекста задачи)
- `Executor` (для понимания, кто выполнил задачу)
- `Created time` (для понимания давности задачи)
- `Last edited time` (для понимания последней активности)

**Опциональные поля:**
- `RFC Link` (для быстрого доступа к связанным RFC)
- `Upstream Source` (для понимания источника задачи)
- `Requires Review` (для подтверждения необходимости ревью)

#### 2. Сортировка

**Основная сортировка:**
- По `SLA` (ascending) — сначала задачи с ближайшим сроком ревью

**Вторичная сортировка:**
- По `Overdue` (descending) — просроченные задачи вверху
- По `Created time` (ascending) — старые задачи вверху

#### 3. Фильтры

**Базовые фильтры (уже настроены):**
- `Status` is `Review`
- `Requires Review` is `Checked`

**Дополнительные фильтры:**
- `Overdue` is `Yes` (для выделения просроченных ревью)
- `Lane` is one of {Infra, Docs, IA, Content, Stories} (для фокуса на конкретных дорожках)
- `SLA` is not empty (для исключения задач без SLA)

#### 4. Группировка (опционально)

Если нужна группировка:
- Group by: `Lane` (для организации по дорожкам)
- Или Group by: `Overdue` (для разделения просроченных и текущих)

#### 5. Визуальные улучшения

- **Цветовое кодирование строк:**
  - Красный фон — просроченные ревью (`Overdue = Yes`)
  - Желтый фон — ревью с приближающимся SLA (менее 12 часов)
  - Белый фон — обычные ревью

- **Иконки:**
  - Просроченные ревью — красный восклицательный знак
  - Приближающийся SLA — желтый предупреждающий знак

## Представление 3: Ready for Copilot (Table)

### Текущее состояние

**Примечание**: В текущей документации есть представление `Ready for CodeGPT`, но нет `Ready for Copilot`. Рекомендуется создать отдельное представление для Copilot.

- **Тип**: Table (новое представление)
- **Фильтры**: `Status = Ready` AND `Executor = Cursor` (или `Executor` contains `Copilot`)

### Рекомендуемые улучшения

#### 1. Видимые поля (Properties)

**Обязательные поля:**
- `Title` (всегда видим)
- `Status` (для подтверждения статуса Ready)
- `Executor` (для подтверждения, что задача для Copilot)
- `Lane` (для понимания контекста задачи)
- `SLA` (для контроля сроков)
- `Created time` (для понимания давности задачи)
- `Upstream Source` (для понимания источника задачи)

**Опциональные поля:**
- `RFC Link` (для быстрого доступа к связанным RFC)
- `Requires Review` (для понимания необходимости ревью после выполнения)
- `Overdue` (для контроля просроченных задач)

#### 2. Сортировка

**Основная сортировка:**
- По `SLA` (ascending) — сначала задачи с ближайшим сроком

**Вторичная сортировка:**
- По `Lane` (ascending) — группировка по дорожкам
- По `Created time` (ascending) — старые задачи вверху

#### 3. Фильтры

**Базовые фильтры:**
- `Status` is `Ready`
- `Executor` is `Cursor` OR `Executor` contains `Copilot`

**Дополнительные фильтры:**
- `Lane` is one of {Infra, Docs, IA, Content, Stories} (для фокуса на конкретных дорожках)
- `SLA` is not empty (для исключения задач без SLA)
- `Overdue` is `No` (для исключения просроченных задач, если они не должны отображаться)

#### 4. Группировка (опционально)

Если нужна группировка:
- Group by: `Lane` (для организации по дорожкам)
- Или Group by: `Executor` (для разделения Cursor и Copilot)

#### 5. Визуальные улучшения

- **Цветовое кодирование строк:**
  - Зеленый фон — задачи готовы к выполнению
  - Желтый фон — задачи с приближающимся SLA (менее 24 часов)
  - Красный фон — просроченные задачи (`Overdue = Yes`)

- **Иконки:**
  - Готовые задачи — зеленая галочка
  - Приближающийся SLA — желтый предупреждающий знак
  - Просроченные задачи — красный восклицательный знак

## Общие рекомендации для всех представлений

### 1. Консистентность

- Использовать одинаковые видимые поля там, где это уместно
- Применять единое цветовое кодирование для статусов и приоритетов
- Соблюдать единый стиль сортировки (SLA в первую очередь)

### 2. Производительность

- Ограничить количество видимых полей (не более 8-10)
- Использовать фильтры для уменьшения количества отображаемых записей
- Избегать сложных формул в видимых полях

### 3. Доступность

- Использовать понятные названия полей
- Добавлять описания к представлениям (Description в Notion)
- Использовать иконки и цвета для быстрого визуального сканирования

### 4. Обновление документации

После внесения изменений обновить:
- `docs/briefs-upgrade-how-to.md` — добавить инструкции по настройке новых представлений
- `docs/protocol-kontraktnaya-model-dlya-agentov.md` — обновить описание представлений

## Инструкция по применению

### Шаг 1: Обновление представления By Lane

1. Откройте представление `By Lane` в Notion
2. Нажмите на `...` (три точки) в правом верхнем углу представления
3. Выберите `Properties` → настройте видимые поля согласно рекомендациям
4. Выберите `Sort` → настройте сортировку по `SLA` (ascending)
5. Выберите `Filter` → добавьте фильтры для исключения `Done` и `Cancelled`
6. Сохраните изменения

### Шаг 2: Обновление представления Pending Review

1. Откройте представление `Pending Review` в Notion
2. Нажмите на `...` (три точки) в правом верхнем углу представления
3. Выберите `Properties` → настройте видимые поля согласно рекомендациям
4. Выберите `Sort` → настройте сортировку по `SLA` (ascending), затем по `Overdue` (descending)
5. Выберите `Filter` → проверьте существующие фильтры и добавьте дополнительные при необходимости
6. Сохраните изменения

### Шаг 3: Создание представления Ready for Copilot

1. Нажмите `+ Add a view` в верхней части базы данных Briefs
2. Выберите тип `Table`
3. Назовите представление `Ready for Copilot`
4. Добавьте фильтры:
   - `Status` is `Ready`
   - `Executor` is `Cursor` OR `Executor` contains `Copilot`
5. Настройте видимые поля согласно рекомендациям
6. Настройте сортировку по `SLA` (ascending)
7. Сохраните представление

### Шаг 4: Визуальные улучшения

1. Для каждого представления настройте цветовое кодирование через условное форматирование (если поддерживается Notion)
2. Добавьте иконки к статусам через кастомные свойства или формулы (если необходимо)
3. Добавьте описания к представлениям через `Description` в настройках представления
