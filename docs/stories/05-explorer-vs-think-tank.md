# Explorer против Think Tank: делим витрину и лабораторию

TL;DR

- В README появился блок Think Tank vs Labs/Explorer.
- Think Tank отвечает за стратегию, Explorer — за кейсы и Stories.
- Бейджи Docs CI / Pages / Release показывают прозрачность процессов.
- `build-index.mjs` и `app.js` автоматически скрывают service-файлы.
- Следующий шаг — наполнить Explorer живыми историями.

**Что произошло.** В README появился блок «Навигация: Think Tank vs Labs/Explorer». Это не декор, а новая логика. Think Tank хранит ADR и архитектуру, Explorer показывает карточки услуг, макеты и теперь Stories.

**Зачем это делали.** Читатель должен сразу понимать, где стратегия, а где практические кейсы. Telegram-подписчики идут в Explorer, а партнёры и подрядчики могут углубиться в Think Tank, если хотят причину решений.

**Что получилось.** Бейджи Docs CI / Pages / Release закрепили прозрачность процессов, а ссылка на Pages ведёт на живой прототип. Он пока статичен, но уже разделяет материалы по статусу и отдаёт summaries.

**Тех-вставка.** `prototype/index.html` использует фильтры и поиск, а `app.js` подтягивает `data/pages.json`, который собирает `build-index.mjs`. Всё, что имеет `service: true`, автоматически скрывается из Explorer, поэтому Stories появятся там же без дополнительных условий.

**Что дальше.** Чтобы эта структура ожила, нужно собственно наполнить Explorer сюжетами. Следующий шаг — выпуск серии и подключение её к Pages.
