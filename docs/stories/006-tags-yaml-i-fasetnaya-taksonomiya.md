# tags.yaml и фасетная таксономия

TL;DR

- Создана система фасетов: theme/*, action/*, product/*, tool/*, role/*, country/*, producer/*.
- Видимые хэштеги используют TitleCase с подчёркиваниями, машинотеги скрыты в front matter.
- Файл tags.yaml содержит маппинг aliases → canonical для автоматической генерации machine_tags.
- Разделение видимых и скрытых тегов исключает визуальную путаницу и упрощает поиск.
- Дальше — автоматизация обогащения тегов через normalize и проверка покрытия алиасов.

**Что произошло.** После нормализации потребовалась система для организации тегов и обеспечения консистентности навигации. Создана фасетная таксономия с разделением на видимые хэштеги для пользователей и машинотеги для автоматизации. Файл `docs/nav/tags.yaml` содержит маппинг алиасов на канонические значения.

**Зачем это делали.** Нужна была система, которая исключает визуальную путаницу и упрощает поиск. Разделение видимых и скрытых тегов позволяет показывать пользователям понятные ярлыки, а машинам — структурированные фасеты. Фасет country/* покрывает кейсы флагов и стран без дублирования тегов.

**Что получилось.** Определены фасеты: theme/* для тем, action/* для действий, product/* для продуктов, tool/* для инструментов, role/* для ролей, country/* для стран, producer/* для производителей. Видимые хэштеги используют TitleCase с подчёркиваниями, машинотеги живут в front matter и скрыты от UI. Файл tags.yaml содержит маппинг aliases → canonical.

**Тех-вставка.** ADR описан в `docs/adr-tegi-i-fasety.md`. В normalize добавлена генерация machine_tags по aliases из tags.yaml. В UI не показываем слаги типа theme/ml, только видимые теги. Покрытие алиасов отслеживается через отчёты: на момент v1.0 покрыто 26 из 40 алиасов (65%).

**Что дальше.** После создания таксономии следующий шаг — автоматизация обогащения тегов через normalize и проверка покрытия алиасов. Это позволит автоматически генерировать machine_tags и обеспечивать консистентность навигации.
