# Автопилот: Notion → Cursor → GitHub

TL;DR

- Дмитрий устал жать «Merge», и мы сняли лишнюю защиту ветки.
- Автопилот читает Briefs, создаёт ветку, ждёт Docs CI и мержит сам.
- Карточка Briefs обновляется автоматически после merge.
- Токен с нужными правами хранится в `.cursor/secrets.json`, sanity-check фиксируем в PR.
- Теперь можно сосредоточиться на витрине и новой аудитории.

**Что произошло.** Дмитрий устал жать «Merge» вручную. Мы упростили branch protection: оставили только Docs CI в Required checks и обнулили approvals. Теперь Cursor берёт задачу из Briefs, делает ветку и мержит сам.

**Зачем это делали.** Проект идёт в темпе, а человеческое подтверждение нужно только на уровне содержания. Автопилот освобождает Дмитрия: он формирует бриф, а после merge получает уже готовый результат и проверяет только смысл.

**Что получилось.** В инструкциях Cursor прописано: зелёный CI — сразу merge, затем обновить карточку Briefs (Deliverables, PR link, Status=Done). Полный цикл — от Ready до Done — занимает пару часов без простоя.

**Тех-вставка.** Токен, лежащий в `C:\Users\utemix\.cursor\secrets.json`, имеет права push и merge. Мы задокументировали sanity-check в PR: API `/branches/main/protection` возвращает 404 без admin scope, поэтому подтверждаем текстом, что включён только Docs CI.

**Что дальше.** Когда рутина снята, можно заняться витриной. Следующий эпизод — как Explorer отделили от Think Tank, чтобы показать истории наружу.
