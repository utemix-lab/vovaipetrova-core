# Stories: daily cadence и генератор

TL;DR

- Создан скрипт generate-stories.mjs для автоматической генерации эпизодов Stories.
- Генератор читает CHANGELOG.md, ADR, stats.json и Briefs для извлечения фактов.
- Workflow stories.yml запускается ежедневно в 07:00 Europe/Moscow и создаёт один draft в день.
- Генератор проверяет наличие эпизода за день и пропускает создание, если фактов нет.
- Дальше — развитие генератора и улучшение качества автоматически создаваемых эпизодов.

**Что произошло.** После создания пилотной серии потребовалась автоматизация для поддержания регулярного потока Stories. Создан скрипт `scripts/generate-stories.mjs` для автоматической генерации эпизодов на основе CHANGELOG, ADR, stats.json и Briefs. Workflow запускается ежедневно и создаёт один draft в день.

**Зачем это делали.** Нужен был регулярный поток Stories без ручного создания каждого эпизода. Генератор автоматически извлекает факты из артефактов проекта и создаёт структурированные эпизоды по шаблону. Это позволяет поддерживать хронику проекта автоматически.

**Что получилось.** Генератор читает CHANGELOG.md, ADR файлы, prototype/data/stats.json и Briefs для извлечения фактов. Создаёт эпизод с front matter из templates/story.md, соблюдает структуру и объём 700-1200 знаков. Workflow stories.yml запускается ежедневно в 07:00 Europe/Moscow, создаёт ветку и PR с меткой auto:story.

**Тех-вставка.** Скрипт описан в `scripts/generate-stories.mjs`, workflow в `.github/workflows/stories.yml`. Генератор проверяет наличие эпизода за день через формат `YYYY-MM-DD-<slug>.md` и пропускает создание, если фактов нет. PR создаётся с заголовком "chore: story auto-YYYY-MM-DD (draft)" и меткой auto:story.

**Что дальше.** После создания генератора следующий шаг — развитие системы и улучшение качества автоматически создаваемых эпизодов. Это позволит поддерживать регулярный поток Stories и улучшать качество контента.
