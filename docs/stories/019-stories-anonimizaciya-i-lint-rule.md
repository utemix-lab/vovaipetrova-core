# Stories: анонимизация и lint-rule

TL;DR

- Добавлено lint-правило для проверки персоналий в stories/ файлах.
- Правило предупреждает о словах: "Дмитрий", "я", "меня", "мне", "мой/моя/мои", "я считаю/думаю/хочу", "по-моему".
- Разрешено использование "мы" в stories для нейтрального повествования.
- Шаблон story.md содержит примечание об анонимизации.
- Дальше — поддержание нейтрального стиля и развитие правил по мере необходимости.

**Что произошло.** После создания пилотной серии потребовалось обеспечить нейтральный стиль повествования без персоналий. Добавлено lint-правило в `scripts/lint-docs.mjs` для проверки stories/ файлов на наличие запрещённых фраз. Правило предупреждает, но не блокирует сборку.

**Зачем это делали.** Нужен был нейтральный стиль для публичных Stories без упоминания конкретных имён. Правило помогает авторам соблюдать стиль и избегать персоналий, сохраняя при этом возможность использовать "мы" для коллективного повествования.

**Что получилось.** Добавлена функция `containsForbiddenStoryPhrases()` для проверки stories/ файлов. Правило предупреждает о словах: "Дмитрий", "я", "меня", "мне", "мой/моя/мои", "я считаю/думаю/хочу", "по-моему". Разрешено использование "мы" для нейтрального повествования. Шаблон содержит примечание об анонимизации.

**Тех-вставка.** Правило реализовано в `scripts/lint-docs.mjs` через проверку пути файла на `stories/` и функцию `containsForbiddenStoryPhrases()`. Предупреждение: "Истории ведём от нейтрального автора; используйте «автор» или безличные формулировки". Правило не блокирует сборку, только предупреждает.

**Что дальше.** После внедрения правил следующий шаг — поддержание нейтрального стиля и развитие правил по мере необходимости. Это позволит сохранить консистентность стиля и улучшить качество Stories.
