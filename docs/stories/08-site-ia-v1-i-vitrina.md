# Site IA v1: готовим витрину

TL;DR

- Появился `docs/nav/routes.yml` с маршрутами будущего сайта.
- Навигация Think Tank ↔ Labs/Explorer описана прямо в README.
- Даже без полноценного сайта Pages знает, что показывать посетителю.
- Workflow pages.yml следит за prototype/** и docs/**, поэтому Stories автоматически попадут в сборку.
- Дальше — релиз пилотной серии и ссылка на раздел Stories.

**Что произошло.** Мы собрали файл `docs/nav/routes.yml`, который описывает разделы `/`, `/project`, `/think-tank`, `/kb`, `/portfolio`, `/nav`. Хотя файл ещё молодой, он уже показывает, какие документы обслуживают каждый маршрут.

**Зачем это делали.** Stories должны появиться на Pages как отдельный раздел Explorer. Без карты маршрутов непонятно, куда вешать ссылки и как потом масштабировать меню. IA v1 — это база для будущего сайта, а не просто вспомогательный YAML.

**Что получилось.** Помимо `routes.yml` появилась памятка `docs/nav/README.md` (сейчас временно убрана, но содержание перенесено в контент-модель). В README репозитория отдельным блоком описана навигация Think Tank ↔ Labs/Explorer, плюс добавлена ссылка на Pages.

**Тех-вставка.** Workflow `pages.yml` запускается при изменениях в `prototype/**`, `docs/**`, `package*.json`. Значит, Stories достаточно добавить в `docs/stories/`, и они автоматически попадут в `pages.json`, а затем в артефакт GitHub Pages.

**Что дальше.** На этой базе мы и разворачиваем пилотную серию: создаём сами истории, добавляем раздел Explorer и показываем ссылку в PR.
