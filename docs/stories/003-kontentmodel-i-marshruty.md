# Контент-модель и маршруты

TL;DR

- Определены основные разделы сайта и их корневые URL.
- Карта маршрутов учитывает корзину артефактов и заявки.
- Роль пользователя влияет на подсказки и приоритеты навигации.
- Документ стал основой для генерации sitemap и линтера.
- Дальше — реализация навигационных компонентов в прототипе.

**Что произошло.** После формирования структуры Think Tank потребовалось определить, как страницы сайта будут организованы и какие URL они обслуживают. Создана карта контент-модели с учётом разделов, ролей пользователей и функциональности корзины артефактов.

**Зачем это делали.** Нужна была основа для генерации sitemap, линтера и навигационных компонентов. Карта маршрутов показывает, что видит каждая роль и какие URL обслуживает система. Это позволяет проектировать UI с учётом пользовательских путей и технических ограничений.

**Что получилось.** Определены основные разделы: главная, услуги, портфолио, база знаний, Think Tank, студия, авторизация. Карта учитывает артефакты и заявки: список артефактов, шторка выбора, форма заявки. Роль пользователя настраивает подсказки и быстрые кнопки через параметр в cookie или сессии.

**Тех-вставка.** Контент-модель описана в `docs/kontentmodel-i-marshruty.md`. Компоненты UI документируются в `prototype/components/*`. Навигационные блоки синхронизируются с `navigaciya-*.md`. Документ является основой для генерации sitemap и проверки ссылок линтером.

**Что дальше.** После определения структуры следующий шаг — реализация навигационных компонентов в прототипе. Это позволит проверить концепцию на практике и уточнить детали взаимодействия.
