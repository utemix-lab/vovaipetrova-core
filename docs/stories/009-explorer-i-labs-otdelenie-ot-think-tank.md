# Explorer и Labs: отделение от Think Tank

TL;DR

- Explorer/Labs выделены как публичная витрина для показа контента наружу.
- Think Tank остаётся внутренней архитектурой с ADR и процессами.
- Explorer получает фильтры, сортировки и карточки для навигации по docs/.
- Разделение позволяет показывать истории и контент публично, сохраняя архитектуру приватной.
- Дальше — развитие Explorer с новыми возможностями навигации и визуализации.

**Что произошло.** После формирования Think Tank потребовалась публичная витрина для показа контента наружу. Explorer/Labs выделены как отдельный раздел для визуализации docs/ с фильтрами, сортировками и карточками. Think Tank остаётся внутренней архитектурой с ADR и процессами.

**Зачем это делали.** Нужна была возможность показывать истории и контент публично, сохраняя архитектурные решения приватными. Explorer позволяет пользователям просматривать контент через удобный интерфейс, а Think Tank остаётся источником истины для разработчиков и процессов.

**Что получилось.** Explorer получает фильтры по статусу, сортировки по route и status, карточки с summary и тегами. Think Tank остаётся в docs/think-tank-* с ADR и процессами. Разделение позволяет показывать публично только готовый контент, скрывая внутреннюю архитектуру.

**Тех-вставка.** Explorer описан в README как Labs/Explorer с прототипом на GitHub Pages. Фильтры и сортировки реализованы в `prototype/app.js`, карточки рендерятся через `createCard()`. Think Tank документирован в `docs/think-tank-kompaktnoe-yadro.md` и `docs/think-tank-vitrina-processa.md`.

**Что дальше.** После разделения следующий шаг — развитие Explorer с новыми возможностями навигации и визуализации. Это позволит улучшить пользовательский опыт и сделать контент более доступным.
