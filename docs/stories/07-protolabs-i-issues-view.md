# ProtoLabs: диагностика и Issues View

TL;DR

- Explorer превратился в ProtoLabs с панелью Issues.
- Теперь проблемы видны без логов: бейджи показывают типы ссылочных ошибок.
- `app.js` рендерит карточки и таблицу «топ проблем».
- `build-index.mjs` создаёт pages.json, а отдельные скрипты — broken-links и stats.
- Следующая задача — Stories как отдельная лента.

**Что произошло.** Explorer вырос в ProtoLabs: помимо карточек появилась панель Issues. Мы научились считать количество внешних, сервисных и внутренних ссылочных проблем прямо в интерфейсе.

**Зачем это делали.** До этого статистику приходилось собирать вручную из логов. Теперь менеджер открывает ProtoLabs и видит, какие страницы тянут проект вниз, а какие уже готовы к публикации.

**Что получилось.** `app.js` теперь рендерит не только карточки, но и список «топ проблем», фильтр по статусам и метрики на каждой карточке. Красные бейджи показывают, сколько именно внутренних ссылок ещё нужно починить.

**Тех-вставка.** `build-index.mjs` генерирует `prototype/data/pages.json`, а соседние скрипты выводят `broken-links.json` и `stats.json`. Эти файлы подхватываются фронтендом и позволяют работать офлайн, что особенно удобно для Pages.

**Что дальше.** Следующий слой — отдельная лента Stories. Она станет ещё одной вкладкой ProtoLabs, но уже для читателей вне команды.
