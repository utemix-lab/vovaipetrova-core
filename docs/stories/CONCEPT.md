# Stories — Концепция и принципы

## TL;DR

Stories — это короткие эпизоды (700–1200 знаков), фиксирующие ключевые вехи развития проекта. Формат: что произошло → зачем → что получилось → тех-вставка → что дальше. Stories создают живую хронику проекта и объясняют контекст решений.

## Философия Stories

### Зачем нужны Stories

1. **Мост между Telegram и ADR**: Stories связывают публичные публикации с внутренними архитектурными решениями
2. **Объяснение контекста**: Каждый эпизод объясняет, почему были нужны те или иные скрипты, прототипы, решения
3. **Живая хроника**: Stories фиксируют развитие проекта в реальном времени
4. **Онбординг**: Новые участники могут быстро понять историю проекта через Stories

### Принципы

- **Краткость**: 700–1200 знаков — достаточно для сути, но не перегружает
- **Структура**: Единый формат для всех эпизодов (см. шаблон)
- **Нейтральность**: Пишем от нейтрального автора, без персоналий
- **Фактологичность**: Фиксируем факты, решения и результаты
- **Техническая точность**: Тех-вставка должна быть точной и полезной

## Структура эпизода

Каждый эпизод Stories следует единой структуре:

1. **TL;DR** — краткое резюме (3–5 пунктов)
2. **Что произошло** — описание события или решения
3. **Зачем это делали** — мотивация и цели
4. **Что получилось** — результаты и артефакты
5. **Тех-вставка** — технические детали (2–3 предложения)
6. **Что дальше** — следующие шаги или планы

## Формат файла

### Front matter

```yaml
---
title: "Название эпизода"
slug: "XXX-краткое-описание"
summary: "Краткая аннотация (1–2 предложения)"
tags: [Story]
machine_tags: [content/story]
status: draft  # draft | review | ready
last_edited_time: null
---
```

### Статусы

- **draft** — черновик, требует доработки
- **review** — готов к ревью
- **ready** — готов к публикации

## Процесс создания Stories

### Ручное создание

1. Используйте шаблон `templates/story.md`
2. Заполните структуру эпизода
3. Проверьте соответствие формату (700–1200 знаков)
4. Запустите `npm run pii:scan` для проверки PII
5. Создайте PR с меткой `content/story`

### Автоматическая генерация

Генератор `scripts/generate-stories.mjs` создаёт эпизоды автоматически:

- Читает CHANGELOG.md, ADR, stats.json и Briefs
- Извлекает факты и события
- Создаёт эпизод по шаблону
- Запускается ежедневно через workflow

## Интеграция в проект

### Explorer

Stories интегрированы в Explorer через `prototype/build-index.mjs`:

- Поле `collection: "stories"` в front matter
- Поле `story_order` из slug для сортировки
- Фильтрация через функцию `isStoryPage()`
- Отображение в ленте Stories на сайте

## Правила работы

### Safety Rails

- ❌ Не использовать первое лицо с именами реальных людей
- ✅ Использовать анонимные плейсхолдеры: `[Имя]`, `[Email]`
- ✅ Проверка PII обязательна перед PR (`npm run pii:scan`)
- ✅ Соблюдать структуру и формат эпизода

### QA-ворота

Перед PR:
1. Проверка формата (700–1200 знаков)
2. Проверка структуры (TL;DR, что произошло, зачем, что получилось, тех-вставка, что дальше)
3. Проверка PII (`npm run pii:scan`)
4. Проверка линтинга (`npm run lint:docs`)
