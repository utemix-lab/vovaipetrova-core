# Site IA v1.1 и routes.yml

TL;DR

- Создан файл routes.yml с картой маршрутов сайта и маппингом на docs/.
- Определены основные разделы: /, /project, /think-tank, /kb, /portfolio, /nav.
- Маршруты сопоставлены с реальными документами, отмечены missing для отсутствующих.
- Explorer получил сортировку "By route" по умолчанию с использованием routes.json.
- Дальше — развитие IA и добавление новых маршрутов по мере роста контента.

**Что произошло.** После формирования контентного ядра потребовалась формализация информационной архитектуры сайта. Создан файл `docs/nav/routes.yml` с картой маршрутов и маппингом на реальные документы в docs/. Определены основные разделы и их соответствия.

**Зачем это делали.** Нужна была основа для навигации и понимания структуры сайта. Карта маршрутов показывает, какие URL обслуживает система и какие документы им соответствуют. Это позволяет планировать развитие контента и обеспечивать консистентность навигации.

**Что получилось.** Определены основные разделы: главная (/), проект (/project), Think Tank (/think-tank), база знаний (/kb), портфолио (/portfolio), навигация (/nav). Маршруты сопоставлены с реальными документами, отмечены missing для отсутствующих. Explorer получил сортировку "By route" по умолчанию через routes.json.

**Тех-вставка.** Файл routes.yml описан в `docs/nav/navigaciya-shema-razdelov-i-marshrutov.md`. Скрипт `scripts/generate-routes-json.mjs` генерирует routes.json для использования в Explorer. Сортировка "By route" реализована в `prototype/app.js` через функцию `sortByRoute()` с использованием routeMap.

**Что дальше.** После формализации IA следующий шаг — развитие архитектуры и добавление новых маршрутов по мере роста контента. Это позволит поддерживать актуальность структуры и обеспечивать удобную навигацию.
