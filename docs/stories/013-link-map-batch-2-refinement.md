# Link-map batch-2: refinement правил

TL;DR

- Расширены правила link-map для обработки следующего набора проблемных ссылок.
- Добавлены mappings для новых service-файлов и паттернов.
- Обработаны ссылки из topProblems из stats.json для снижения internal-missing.
- Система позволила резолвить больше ссылок без изменения контента docs/.
- Дальше — дальнейшее расширение правил и обработка оставшихся проблемных ссылок.

**Что произошло.** После batch-1 остались проблемные ссылки, которые требовали дополнительных правил. Расширены правила link-map для обработки следующего набора ссылок из topProblems в stats.json. Добавлены mappings для новых случаев и уточнены существующие паттерны.

**Зачем это делали.** Нужно было снизить количество internal-missing ссылок без изменения контента docs/. Расширение правил позволяет резолвить больше ссылок автоматически, улучшая качество навигации и снижая количество broken links в отчётах.

**Что получилось.** Добавлены mappings для новых service-файлов и уточнены паттерны. Обработаны ссылки из topProblems, что снизило количество internal-missing. Система позволила резолвить больше ссылок без правок в исходных файлах, улучшив качество навигации.

**Тех-вставка.** Расширение правил описано в PR "chore: site IA v1.1 + explorer defaults + link-map refine (batch-2)". Правила добавлены в `prototype/link-map.json`, статистика обновлена через `scripts/generate-stats.mjs`. Результаты отражены в `prototype/data/stats.json` и `broken-links.json`.

**Что дальше.** После batch-2 следующий шаг — дальнейшее расширение правил и обработка оставшихся проблемных ссылок. Это позволит довести internal-missing до нуля и улучшить качество навигации.
