# Версия v1.0: фиксируем контентное ядро

TL;DR

- 12 ноября поставлен тег `v1.0-content-ready`.
- 24 страницы готовы к публикации, 105 оформлены как внимательные черновики.
- README и CHANGELOG получили статистику и список сделанных шагов.
- Скрипты report-content-stats и release API автоматизировали фиксацию.
- Теперь каждое новое изменение сравнивается с понятным baseline.

**Что произошло.** 12 ноября мы поставили тег `v1.0-content-ready`. Это первый момент, когда можно честно сказать: «Контентная база собрана, структура понятна, линтеры молчат».

**Зачем это делали.** Цель — иметь точку отсчёта. 24 ключевых страницы получили статус `ready`, ещё 105 оформлены как `draft` с summary и тегами. Теперь любой следующий пакет измеряется относительно этой версии.

**Что получилось.** README обновился разделом «Статистика контента»: 154 файла, 0 предупреждений линтера, покрытие алиасов 65 %. CHANGELOG зафиксировал эти цифры, а release note описал, какие скрипты доведены до стабильного состояния.

**Тех-вставка.** Скрипт `scripts/report-content-stats.mjs` собирает JSON и Markdown-выгрузку, а release создаётся через GitHub API с телом, в котором перечислены Highlights и Checks. Pages деплоится по workflow `pages.yml`, так что публикация Stories упрётся в тот же pipeline.

**Что дальше.** После контрольной точки можно экспериментировать с витриной: добавлять Stories, Issues view, навигацию. Всё новое сразу видно на фоне v1.0, и это снижает страх правок.
