# [ADR] Журнал решений

## TL;DR
- Журнал фиксирует все стратегические решения по архитектуре и процессам.
- Каждая запись — ссылка на отдельную страницу ADR с контекстом, вариантами и последствиями.
- Статусы ADR помогают отличать принятые решения от исследуемых или отменённых.

## Назначение журнала
- Держать единый список активных решений, чтобы команда видела историю изменений.
- Упростить аудит: по ID легко найти обсуждение и PR, в котором внедряли решение.
- Помогать новым участникам быстро понять действующие ограничения.

## Формат записи
| Поле       | Описание                                     |
| ---------- | --------------------------------------------- |
| `ID`       | Последовательный номер (`ADR-0005`)           |
| `Дата`     | Когда решение принято или обновлено           |
| `Статус`   | Proposed → Accepted → Superseded → Rejected   |
| `Ссылка`   | Путь к документу ADR в репозитории            |
| `Кратко`   | 1 предложение о сути решения                  |

## Как вести журнал
1. Создаём ADR-шаблон из `templates/adr.md` и описываем контекст, варианты, последствия.  
2. Ставим ссылку на связанные файлы и PR, где решение реализовано.  
3. Добавляем строку в таблицу журнала, обновляем статус при изменениях.
