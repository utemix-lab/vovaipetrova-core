# TEMPLATE — Task spec Example 3: Прототип UI

**Тип задачи**: Улучшение UI прототипа
**Для**: Composer (Cursor AI Agent)
**Дата**: 2025-11-20

---

## Обязательные поля

### Что нужно сделать (результат)

**Краткое описание**: Добавить фильтр по статусу (`ready`, `review`, `draft`) в Explorer UI (`prototype/index.html`) и обновить стили для лучшей визуализации статусов.

### Где менять

**Разрешённые пути**:
- `prototype/index.html` — добавить UI фильтра по статусу
- `prototype/app.js` — добавить логику фильтрации
- `prototype/styles.css` — добавить стили для статусов

**Запрещённые пути** (safety rails):
- ❌ `prototype/data/*.json` — данные генерируются автоматически, не менять
- ❌ `prototype/page/*.html` — генерируемые страницы, не трогать

### Требования

**Функциональность**:
- Добавить dropdown/чекбоксы для фильтрации по статусу (`ready`, `review`, `draft`)
- Фильтр должен работать в реальном времени (без перезагрузки)
- Сохранять выбранные фильтры в `localStorage` для удобства пользователя
- Показывать количество страниц по каждому статусу

**UI/UX**:
- Фильтр должен быть видимым и доступным в верхней части Explorer
- Визуальные индикаторы статусов (цветные бейджи/метки)
- Адаптивный дизайн (работает на мобильных устройствах)

**Стиль кода**:
- Использовать существующие паттерны из `app.js`
- Комментарии для сложной логики
- Соответствие стилю существующего кода

### Safety Rails

**Запрещённые действия**:
- ❌ Не изменять структуру данных в `prototype/data/*.json`
- ❌ Не добавлять внешние зависимости без согласования
- ❌ Не изменять существующую функциональность без явного указания

**Проверки перед PR**:
1. ✅ `npm run normalize:dry` — проверить изменения (если есть изменения в `docs/`)
2. ✅ `npm run lint:docs` — проверить качество (если есть изменения в `docs/`)
3. ✅ `npm run check:pr-size` — проверить размер PR
4. ✅ `npm run check:lanes` — проверить lanes policy (должен быть `lane:prototype`)
5. ✅ Локально протестировать UI в браузере
6. ✅ Проверить работу на мобильных устройствах (responsive)
